Синтаксис CDK и ООП (Объектно-Ориентированное Программирование)
Когда ты пишешь new s3.Bucket(this, 'MyWebsiteBucket', { ... }), ты используешь классические паттерны ООП, адаптированные под TypeScript.

Давай разберем эту строчку по косточкам:

s3.Bucket — Это Класс (Class). Чертеж того, как должна выглядеть корзина в AWS. Этот класс написали разработчики Amazon.

new — Ключевое слово, которое вызывает Конструктор (Constructor) этого класса, чтобы создать конкретный объект (экземпляр) по этому чертежу.

В конструктор мы всегда передаем строго 3 аргумента:

this (Scope / Область видимости): Ты говоришь корзине: "Твой родитель — этот текущий Стек". Это выстраивает иерархию (дерево) ресурсов.

'MyWebsiteBucket' (ID / Идентификатор): Это логическое имя ресурса внутри твоего кода. Оно должно быть уникальным в рамках текущего стека.

{ ... } (Props / Свойства): Это самое интересное. Третий аргумент принимает обычный JavaScript/TypeScript объект. В ООП это называется реализацией Интерфейса (Interface) конфигурации. Вместо того чтобы передавать 20 разных аргументов через запятую (что было бы нечитаемо), ты передаешь один объект, в котором перечисляешь только те настройки, которые хочешь изменить.
