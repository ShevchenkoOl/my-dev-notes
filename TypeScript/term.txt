Interface — это описание формы объекта, он используеться только во время разроботки (В JS после сборки его НЕТ), он говорит:
       - какие поля должны быть;
       - какого они типа.
    Interface используеться при более сложной структуре например:
        interface User {
            id: number,
            name: string,
            isOnline: boolean
        } - тут имя ключей уже заранее зарезервировано и мы не имеем права их менять!
    
    или более простая:
    
        type User = Records<string, number>;


Record - делает тоже самое что и interface, только упрозая запись:
        type CurrencyRates = Record<string, number>; - то есть унас будет неопределённое количество пар ключ-значение, они будут иметь разные имена, но тип данных будет ключ - string, значение - number.

index signature - объект может иметь любые string-ключи.
        Например:
        1️⃣   interface CurrencyRates {   // название ключей в этом обьекте могут быть любые
                    [code: string]: number;         
                    }

        2️⃣ interface CurrencyRates {
                        code: string; // название ключей в этом обьекте должны быть только code
                    }


| - Union Type / Объединение типов, это означает что мы можем задать несколько типов данных например: nubmer | boolean

void - используеться для функций которые ничего не возвращают
