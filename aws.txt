/*------------основные термины и понятия -------------------------*/
SQS (Simple Queue Service) — это полностью управляемая служба очередей сообщений от Amazon Web Services (AWS), которая позволяет разделять и масштабировать микросервисы, распределенные системы и бессерверные приложения.

Amazon EC2 (Elastic Compute Cloud) — это веб-сервис от Amazon, который предоставляет виртуальные серверы (называемые «инстансами») в облаке. Простыми словами, это виртуальные компьютеры.

Amazon S3 (Simple Storage Service) — это один из самы порулярных веб-сервис от Amazon предназначеный для хранения информации (облако). Особенность заключаеться в том, что все данные храняться в виде обьектов (Object Storage, парой ключ: значение), можна хранить практически не ограниченое количество информации, данные автоматически копируються в несколько дата-центров, можно сделать файлы приватными, публичными или настроить доступ по ролям. Интеграция с другими сервисами AWS (например, AWS Lambda, Amazon CloudFront, Amazon EC2 и т.д.).

AWS Lambda — это сервис от Amazon Web Services, который позволяет запускать код без серверов. Проще говоря, Lambda = ты пишешь функцию, а AWS сам заботится о серверах, масштабировании и запуске.
            Как это работает:
                - Ты загружаешь кусочек кода — функцию (например, на Python, Node.js, Go, Java и т.д.).
                - Указываешь, когда она должна запускаться (например, при загрузке файла в Amazon S3, при HTTP-запросе, по расписанию и т.п.).
                - AWS автоматически запускает твою функцию, когда происходит событие.
            Пример:
                    Представь, что ты загружаешь изображение в S3. Можно сделать так, чтобы Lambda автоматически срабатывала и, например, уменьшала размер картинки или добавляла водяной знак.

AWS CloudFormation — это способ описать всю инфраструктуру AWS в одном файле и автоматически её создать.
                    Вместо того чтобы вручную создавать: S3 bucket, Lambda, API Gateway или базы данных, ты описываешь всё в YAML или JSON, и AWS сам создаёт эти ресурсы. Когда ты запускаешь этот template, AWS автоматически создаёт S3 bucket.
                    Как это работает
                            1️⃣ Ты пишешь template (YAML / JSON)
                            2️⃣ Загружаешь его в CloudFormation
                            3️⃣ AWS создаёт Stack (набор ресурсов)

                    Stack может включать:
                        Amazon S3
                        AWS Lambda
                        Amazon API Gateway
                        Amazon DynamoDB

                    🚀 Зачем это нужно. Преимущества:
                            - инфраструктура хранится в Git
                            - можно быстро развернуть одинаковую систему
                            - легко обновлять ресурсы
                            - удобно для DevOps и CI/CD


/*----------------------------Основные команды в AWS CLI (CloudShell)---------------------------*/
Почти все команды строятся по принципу: aws + название сервиса + действие. Это быстрые команды для проверки статуса или разовых действий с ресурсами.

Работа с S3 (хранилище):
            aws s3 ls — показать список всех твоих бакетов.
            aws s3 mb s3://имя-бакета — создать новый бакет (make bucket).
            aws s3 cp myfile.txt s3://имя-бакета/ — скопировать файл с компьютера в облако (copy).
            aws s3 rm s3://имя-бакета/myfile.txt — удалить файл из бакета (remove).

Работа с EC2 (серверы):
            aws ec2 describe-instances — показать список всех твоих виртуальных серверов и их статус.
            aws ec2 start-instances --instance-ids i-12345 — запустить сервер.
            aws ec2 stop-instances --instance-ids i-12345 — остановить сервер (чтобы не капали деньги).

Узнать "кто я":
            aws sts get-caller-identity — очень полезная команда. Она покажет твой ID, Account ID и имя пользователя. Помогает проверить, под тем ли аккаунтом ты залогинен.
какие у меня права:

/*--------------------Команды CDK — это команды для развертывания целых проектов из кода.-------------------*/
aws sts get-caller-identity - проверяет или подключён твой проект к AWS, в ответе должен быть JSON с твоим ID и именем

Чертёж: npx cdk synth --context stage=development --context imageTag=latest - Это абсолютно безопасная команда. Её главная задача — проверить, что в твоем коде нет логических ошибок, и показать, что именно планируется создать.
    --context stage=development — это передача переменной внутрь твоего кода. Эта переменная говорит твоему коду: "Мы сейчас работаем в тестовой среде". Благодаря ей твоя таблица в DynamoDB получила приписку development в названии, а не production.

    --context imageTag=latest: Ваш проект, скорее всего, использует Docker-образы для Lambda-функций. Эта переменная говорит AWS: "Когда будешь обновлять функции, возьми образ с тегом latest (самый свежий)". В рабочей среде (production) там обычно передают конкретную версию, например imageTag=v1.16.0,

Стройка: npx cdk deploy --context stage=development --context imageTag=latest

npx cdk deploy --require-approval never - Пропускает тот самый вопрос Do you wish to deploy these changes (y/n)?. Как работает: Деплой идет сразу. Это часто используют роботы в CI/CD (например, GitHub Actions), так как робот не может нажать кнопку y.
  Если огрвниячені права допуска и через терминал (CLI) не получаеться сделать, можна воспользоваться возможностям GitHub:
      Actions -> All workflow -> deploy_noghtly -> Run workflow -> выбираем ветку которую нам нужно задеплойдить -> Run workflow

npx cdk diff - Показывает разницу между тем, что есть в облаке сейчас, и тем, что ты написал в коде. Как работает: Выводит в терминал зелено-красный список (что добавится, а что удалится) до того, как ты запустишь deploy. Очень полезная команда для самопроверки!

npx cdk destroy - Удаляет всё, что ты построил. Как работает: Сносит инфраструктуру (таблицы, очереди и т.д.), описанную в коде, из AWS. С ней нужно быть очень аккуратным!
