/*------------основные термины, понятия и название ресурсов AWS -------------------------*/
AWS CDK (Cloud Development Kit) — это переводчик. Он позволяет тебе создавать облачную инфраструктуру (серверы, базы данных, сети), используя привычный тебе язык программирования — например TypeScript.

SQS (Simple Queue Service) — это полностью управляемая служба очередей сообщений от Amazon Web Services (AWS), которая позволяет разделять и масштабировать микросервисы, распределенные системы и бессерверные приложения.

Amazon EC2 (Elastic Compute Cloud) — это веб-сервис от Amazon, который предоставляет виртуальные серверы (называемые «инстансами») в облаке. Простыми словами, это виртуальные компьютеры.

Amazon S3 (Simple Storage Service) — это один из самы порулярных веб-сервис от Amazon предназначеный для хранения информации (облако). Особенность заключаеться в том, что все данные храняться в виде обьектов (Object Storage, парой ключ: значение), можна хранить практически не ограниченое количество информации, данные автоматически копируються в несколько дата-центров, можно сделать файлы приватными, публичными или настроить доступ по ролям. Интеграция с другими сервисами AWS (например, AWS Lambda, Amazon CloudFront, Amazon EC2 и т.д.).

AWS Lambda — это сервис от Amazon Web Services, который позволяет запускать код без серверов. Проще говоря, Lambda = ты пишешь функцию, а AWS сам заботится о серверах, масштабировании и запуске.
            Как это работает:
                - Ты загружаешь кусочек кода — функцию (например, на Python, Node.js, Go, Java и т.д.).
                - Указываешь, когда она должна запускаться (например, при загрузке файла в Amazon S3, при HTTP-запросе, по расписанию и т.п.).
                - AWS автоматически запускает твою функцию, когда происходит событие.
            Пример:
                    Представь, что ты загружаешь изображение в S3. Можно сделать так, чтобы Lambda автоматически срабатывала и, например, уменьшала размер картинки или добавляла водяной знак.

AWS CloudFront -  это быстрый сервис сети доставки контента (CDN), который ускоряет распространение статических и динамических веб-файлов (изображений, видео, JS, CSS) пользователям по всему миру. Простыми словами, мы его будем использовать для того чтобы раздать наш сайт по безопасному протоколу https://

AWS CloudFormation — это способ описать всю инфраструктуру AWS в одном файле и автоматически её создать.
                    Вместо того чтобы вручную создавать: S3 bucket, Lambda, API Gateway или базы данных, ты описываешь всё в YAML или JSON, и AWS сам создаёт эти ресурсы. Когда ты запускаешь этот template, AWS автоматически создаёт S3 bucket.
                    Как это работает
                            1️⃣ Ты пишешь template (YAML / JSON)
                            2️⃣ Загружаешь его в CloudFormation
                            3️⃣ AWS создаёт Stack (набор ресурсов)

                    Stack может включать:
                        Amazon S3
                        AWS Lambda
                        Amazon API Gateway
                        Amazon DynamoDB

                    🚀 Зачем это нужно. Преимущества:
                            - инфраструктура хранится в Git
                            - можно быстро развернуть одинаковую систему
                            - легко обновлять ресурсы
                            - удобно для DevOps и CI/CD

VPC (Virtual Private Cloud) - сети. Обычно используються для создание серверов (instance)

Aurora RDS (Relational Database Service) — это сервис внутри Амазона специально для управления базами данных. Грубо говоря, это услуга «База данных под ключ». Амазон сам делает бэкапы, сам обновляет версии, сам следит, чтобы база не упала. Тебе не нужно настраивать Linux-сервер для базы, ты просто нажимаешь кнопку «Создать» и получаешь готовый адрес (Endpoint). Aurora— это фирменный движок баз данных от самого Амазона, который работает внутри сервиса RDS. Он полностью совместим с популярными базами (MySQL или PostgreSQL), но работает в несколько раз быстрее и надежнее, так как специально оптимизирован под облако.
            Почему база данных спрятана и зачем нужен туннель?
                - По правилам безопасности (и это стандарт для любой серьезной компании), база данных никогда не должна торчать в открытый интернет.
                - База данных Aurora RDS помещается в Private Subnet (закрытую виртуальную сеть). Никто снаружи не может к ней подключиться напрямую.
                - Чтобы разработчики могли с ней работать, создается Bastion Host (Jump Server) — специальный сервер-проходная, который имеет доступ и в интернет, и в закрытую сеть с базой. В твоем случае этот сервер-проходная имеет IP-адрес 3.120.195.0.
            Как работает твой туннель (разбор команды)
                - Когда ты пишешь свою команду в терминале, ты буквально строишь защищенную трубу от своего компьютера до закрытой базы данных:
                    ssh -A oleksii.shevchenko@3.120.195.0 -L 127.0.0.1:3307:cooking-nightly.ckkrylvymtpv.eu-central-1.rds.amazonaws.com:3306 -N
                        Разбор заклинания:
                            ssh ...@3.120.195.0 — "Подключи меня к серверу-проходной".
                            -L 127.0.0.1:3307 — "Открой на моем локальном компе порт 3307".
                            cooking-nightly...rds.amazonaws.com:3306 — "И скажи серверу-проходной, чтобы весь трафик с моего порта 3307 он перекидывал на порт 3306 этой закрытой базы данных Амазона".
                            -N — "Не открывай мне командную строку сервера, просто держи туннель открытым".
            Шпаргалка: где искать концы, если туннель снова упал
                Если туннель выдает ошибку Name or service not known, это означает, что адрес базы (Endpoint) изменился. В средах вроде Nightly базы часто пересоздаются авто-скриптами, и Амазон генерирует для них новые адреса. Где взять новый адрес в AWS:
                    - Зайти в консоль AWS.
                    - В поиске сервисов вбить RDS.
                    - В левом меню выбрать Databases (Базы данных).
                    - Найти нужную базу (например, cooking-nightly).
                    - Открыть ее и прокрутить до раздела Connectivity & security.
                    - Скопировать значение из поля Endpoint (это и будет тот самый длинный адрес *.rds.amazonaws.com).
                    - Вставить его в свою команду для туннеля.
        📢 <статистичиская часть мы ее берём с файла Readme.md>cooking-nightly.ckkrylvymtpv.eu-central-1.rds.amazonaws.com<:3306 -N> в конце команды мы меняем порт потомучто с вероятностью 99% у нас может бежать доаер на порту 3306. Флаг -N - означает не открывать виртуальный терминал.Целая реальная команда выглядит вот так:
        ssh -A oleksii.shevchenko@3.120.195.0 -L 127.0.0.1:3307:cooking-nightly.ckkrylvymtpv.eu-central-1.rds.amazonaws.com:3306 -N

ssh --vvv (--vvv)- при использование SSH подключения или просто для включения максимально подробного режима логирования (Verbose mode)



/*----------------------------Основные команды в AWS CLI (CloudShell)---------------------------*/
Почти все команды строятся по принципу: aws + название сервиса + действие. Это быстрые команды для проверки статуса или разовых действий с ресурсами.

aws configure в интерфейсе командной строки AWS (AWS CLI) — это основной инструмент для первоначальной настройки подключения к облаку Amazon Web Services
    ‼️ aws configure - это дефолтные настройки, но мы можем отделить рабочме от собственных:
       aws configure --profile personal (--profile — это стандартная команда (флаг) самого AWS, а вот слово personal — это просто имя, которое ты придумываешь сам для своего удобства). Если у меня отделены профили то при использовании дкфолтного я просто ввожу команду aws....., aws [название сервиса] --profile [имя профиля]
    - Где хранятся данные?
        - Введенные данные сохраняются в скрытой папке .aws в домашнем директории пользователя в двух файлах:~/.aws/credentials — здесь в зашифрованном или открытом виде (зависит от настроек ОС) хранятся Access Key ID и Secret Access Key.~/.aws/config — здесь хранятся регион и формат вывода.
    - Полезные флаги:
        aws configure --profile имя_профиля — позволяет создать отдельный профиль для другого проекта или пользователя (например, aws configure --profile dev).
        aws configure list — выводит список текущих активных настроек и пути к файлам конфигурации.


Терминал попросит тебя ввести 4 параметра:
    AWS Access Key ID: скопируй из скачанного файла.
    AWS Secret Access Key: скопируй оттуда же.
    Default region name: введи eu-central-1
    Default output format: введи json

Инициализация проекта:
    npx aws-cdk init app --language typescript     - инициализация нового проекта с использованием языка TypeScript

CDK создаст в твоём AWS специальный служебный S3-бакет и роли, которые ему нужны, чтобы в будущем загружать и разворачивать твой код. Без этого он работать не сможет.
    npx aws-cdk bootstrap

Работа с S3 (хранилище):
            aws s3 ls — показать список всех твоих бакетов.
            aws s3 mb s3://имя-бакета — создать новый бакет (make bucket).
            aws s3 cp myfile.txt s3://имя-бакета/ — скопировать файл с компьютера в облако (copy).
            aws s3 rm s3://имя-бакета/myfile.txt — удалить файл из бакета (remove).

Работа с EC2 (серверы):
            aws ec2 describe-instances — показать список всех твоих виртуальных серверов и их статус.
            aws ec2 start-instances --instance-ids i-12345 — запустить сервер.
            aws ec2 stop-instances --instance-ids i-12345 — остановить сервер (чтобы не капали деньги).

Узнать "кто я":
            aws sts get-caller-identity — очень полезная команда. Она покажет твой ID, Account ID и имя пользователя. Помогает проверить, под тем ли аккаунтом ты залогинен.
какие у меня права:
            aws iam list-attached-user-policies --user-name oleksii.shevchenko


/*--------------------Команды CDK — это команды для развертывания целых проектов из кода.-------------------*/
aws sts get-caller-identity - проверяет или подключён твой проект к AWS, в ответе должен быть JSON с твоим ID и именем

Чертёж: npx cdk synth --context stage=development --context imageTag=latest - Это абсолютно безопасная команда. Её главная задача — проверить, что в твоем коде нет логических ошибок, и показать, что именно планируется создать.
    --context stage=development — это передача переменной внутрь твоего кода. Эта переменная говорит твоему коду: "Мы сейчас работаем в тестовой среде". Благодаря ей твоя таблица в DynamoDB получила приписку development в названии, а не production.

    --context imageTag=latest: Ваш проект, скорее всего, использует Docker-образы для Lambda-функций. Эта переменная говорит AWS: "Когда будешь обновлять функции, возьми образ с тегом latest (самый свежий)". В рабочей среде (production) там обычно передают конкретную версию, например imageTag=v1.16.0,

Стройка: npx cdk deploy --context stage=development --context imageTag=latest

npx cdk deploy --require-approval never - Пропускает тот самый вопрос Do you wish to deploy these changes (y/n)?. Как работает: Деплой идет сразу. Это часто используют роботы в CI/CD (например, GitHub Actions), так как робот не может нажать кнопку y.
  Если огрвниячені права допуска и через терминал (CLI) не получаеться сделать, можна воспользоваться возможностям GitHub:
      Actions -> All workflow -> deploy_noghtly -> Run workflow -> выбираем ветку которую нам нужно задеплойдить -> Run workflow

npx cdk diff - Показывает разницу между тем, что есть в облаке сейчас, и тем, что ты написал в коде. Как работает: Выводит в терминал зелено-красный список (что добавится, а что удалится) до того, как ты запустишь deploy. Очень полезная команда для самопроверки!

npx cdk destroy - Удаляет всё, что ты построил. Как работает: Сносит инфраструктуру (таблицы, очереди и т.д.), описанную в коде, из AWS. С ней нужно быть очень аккуратным!

npm install --save-dev typescript @types/aws-lambda @types/node - Установка зависимостей, в данном случаи для AWS Lambda

Если нам нужно запустить отдельный стек и не на робочем профиле:
npx aws-cdk deploy S3StudyStack --profile personal   S3StudyStack - это название стека
