Золотое Правило Конфигов Economia, которое тебе нужно запомнить для любого следующего микросервиса:

Правило №1: Файлы строго разделены по смыслу.
    - default.ts — Это "Словарь типов" и "Безопасные дефолты". Здесь лежат только те значения, которые не страшно закоммитить в Git (URL-адреса, таймауты, имена таблиц). Здесь же описываются TypeScript-интерфейсы Config и ConfigGeneral. Никаких секретов, никаких process.env!
    - custom-environment-variables.ts — Это "Мост в реальный мир". Этот файл не содержит самих значений, он содержит имена переменных окружения. Он говорит: "Если хочешь узнать ключ базы, ищи его в системной переменной AWS_SECRET_ACCESS_KEY".
    - local.ts / development.ts / production.ts — Это "Переопределения". Эти файлы (особенно local.ts) часто лежат в .gitignore и нужны для локальной разработки, чтобы подменить боевые ссылки на тестовые или локалхост.

Правило №2: Приложение ничего не знает про окружение.
    - В самом коде сервиса (в контроллерах, моделях, сервисах) ты никогда не должен писать:
    - ❌ if (process.env.NODE_ENV === 'production')
    - ❌ const key = process.env.API_KEY
Вместо этого ты пишешь:
✅ const key = getConfig<string>('security.xAuth')
