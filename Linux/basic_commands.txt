1. Назначение операционных систем:
        Windows: ОС общего назначения, ориентированная на графический интерфейс (GUI) и комфорт конечного пользователя.

        Linux: UNIX-подобная ОС, являющаяся индустриальным стандартом для серверов, облачной инфраструктуры, контейнеризации и бэкенд-разработки. Отличается высокой стабильностью, отказоустойчивостью и предоставляет разработчику полный контроль над системой через терминал.

2. Разница CLI (Command Line Interface):
Схожесть команд в терминалах Linux (Bash) и Windows (PowerShell) обманчива:
        - В Linux команды (ls, pwd, rm) — это нативные POSIX-совместимые утилиты.
        - В Windows PowerShell эти же команды являются лишь алиасами (псевдонимами), добавленными для удобства разработчиков. Например, при вводе ls в PowerShell, под капотом выполняется системный командлет Get-ChildItem. Они обращаются к абсолютно разным API операционных систем.

3. Что такое WSL:
   WSL (Windows Subsystem for Linux) — это подсистема, позволяющая запускать полноценное нативное окружение GNU/Linux (например, Debian или Ubuntu) непосредственно внутри Windows, без накладных расходов на классическую виртуальную машину (Dual-boot или VirtualBox).

4. Изоляция файловых систем и распределение памяти:
   Изоляция (VHDX): При установке WSL выделяет память на общем жестком диске в виде единого виртуального диска — файла с расширением .vhdx.

   Разные файловые системы: Windows работает на файловой системе NTFS, а внутри виртуального диска WSL разворачивает файловую систему Linux (обычно ext4).

   Доступ через Проводник: Обычный проводник Windows не умеет читать формат ext4 напрямую, поэтому он видит систему Linux просто как один большой системный файл .vhdx. Чтобы получить доступ к файлам Linux, используется встроенный сетевой мост: путь \\wsl$\, который транслирует файлы из ext4 в понятный для Windows формат.


/*------------------------------------Основные утилиты ---------------------------------------*/
- jq — это мощная и легковесная утилита командной строки, которая используется для парсинга, фильтрации, изменения и форматирования данных в формате JSON. Её часто называют «sed для JSON».
- grep (Global Regular Expression Print) - Она нужна для поиска текста внутри файлов или вывода. Обычно её используют через тот самый пайп (|).
                Пример: cat my_logs.txt | grep "ERROR".
                Флаги которые можна использовать с grep:
                        - grep -i "error" — игнорировать регистр (найдет и ERROR, и Error, и error).
                        - grep -v "INFO" — инвертировать поиск (покажи мне всё, КРОМЕ строк со словом INFO).
                        - grep -A 3 "ERROR" — показать саму ошибку и 3 строки после нее (After), чтобы понять контекст.
                        - grep -B 2 "ERROR" — показать 2 строки до нее (Before).



/*------------------------------- Как настроить собственные алиасы (альтернативное имя или краткая ссылска) -------------------------*/
Алиасы сохраняются в скрытом конфигурационном файле твоего терминала .bashrc
1. nano ~/.bashrc - открываем конфигурационный файл в консольном текстовом редакторе
2. в самом низу файла записываем сохраняем команду которую мы хотим создать альянс: alias <короткая_команда или имя>='<команда в оригинале>'
        alias get-hpx-error='aws logs filter-log-events --log-group-name /aws/lambda/hpx-data-provider-feeder --filter-pattern "{$.level > 30}" --max-items 1000 | jq ".events | map(.message | fromjson)"'
3. сохраняем Ctrl + O -> Enter (подтвердить имя файла) -> Ctrl + X (чтобы выйти из редактора)
4. Для того чтобы терминал прочитал новые настройки, его нужно либо перезапустить, либо выполнить команду:      
        source ~/.bashrc


1️⃣ Навигация и просмотр директорий

pwd              - Выводит абсолютный путь к текущей папке («Где я?»).
cd <папка>       -  Перейти в указанную папку.
cd ..            - Подняться на один уровень выше.
cd ../..         - Подняться на два уровня выше (то есть одна пару точек жто один уровень).
cd ~ (или cd)    - Быстро вернуться в домашнюю директорию пользователя.
cd /повний/шлях  - Перейти по абсолютному пути.
ls               - Показывает имена файлов и папок (только видимые).
ls -la           - Подробный список (-l) всех файлов (-a), включая скрытые (начинаются с .). Показывает права, владельца, размер.
ls -F            - Добавляет символы для различения типов (например, `/` для папок, `*` для исполняемых файлов).
ls -f            - Выводит элементы без сортировки (в порядке хранения на диске). |

👉 Примечание к путям:
        - Если в названии папки есть пробелы, используй кавычки или экранирование: `cd "TASK 2"` или `cd TASK\ 2`.
        - Команда `dir` существует в Linux, но это скорее рудимент для удобства пользователей Windows. В Bash стандартом является `ls`.


2️⃣ Работа с файлами и папками
mkdir <имя>             - Создать новую директорию.
mkdir -p dir1/dir2      - Создать сразу несколько вложенных папок.
touch <файл>            - Создать пустой файл.
cat <файл>              - Вывести содержимое файла в терминал.
cat > <файл>            - Создать файл и сразу записывать в него вводимый текст (выход: `Ctrl+D`).
echo "текст" > <файл>   - Создать файл (или перезаписать) и добавить туда этот текст.
echo "текст" >> <файл>  - Дописать текст в конец существующего файла.
cp <что> <куда>         - Скопировать файл.
cp -r <что> <куда>      - Скопировать директорию рекурсивно (со всем содержимым).
mv <что> <куда>         - Переместить или переименовать файл/папку.
rm <файл>               - Удалить файл (в Linux нет «Корзины», удаляется навсегда!).
rmdir <имя>             - Удалить директорию, **только если она пустая**.
rm -r <имя>             - Рекурсивно удалить папку (спросит подтверждение, если не добавить `-f`).
rm -rf <имя>            - ‼️Опасно: Насильно (`-f`) и рекурсивно (`-r`) удаляет папку и всё её содержимое без вопросов.


3️⃣ Системные команды и утилиты
clear               - Очистить экран терминала.
history             - Показать список последних введенных команд.
sudo <команда>      - Выполнить команду от имени администратора (Superuser DO).
top` (или `htop`)   - Диспетчер задач (посмотреть нагрузку на процессор, память и зависшие процессы).
apt                 - Пакетный менеджер в Debian/Ubuntu (установка, обновление, удаление программ).
which <имя>         - Показывает, где физически установлен исполняемый файл (например, `which node`).

5️⃣ Экосистема Node.js (NVM)
nvm                   — Node Version Manager используется для установки и переключения между разными версиями Node.js.
nvm ls-remote         — показать все версии Node.js, доступные для установки.
nvm ls                — показать версии, установленные локально.
nvm install 24        — установить 24-ю версию Node.js.
nvm alias default 24  — установить версию 24 по умолчанию при открытии терминала.
nvm use               — автоматически применить версию Node.js, указанную в файле `.nvmrc` текущего проекта.


6️⃣ Интеграция WSL, Windows и VS Code
/mnt            — директория в Linux, куда монтируются файловые системы. Например, чтобы перейти на диск D из WSL: cd /mnt/d/<путь к проекту>.
explorer.exe .  — открывает обычный проводник Windows прямо в той директории Linux, где ты сейчас находишься (точка означает текущую папку).
Доступ к файлам WSL из Windows: Если проводник не открывает Linux-окружение, введи в адресной строке проводника Windows: \\wsl$\

👉 Связка VS Code и WSL (Debian):
        1. Открой VS Code в Windows.
        2. Нажми `Ctrl + Shift + P` (командная палитра).
        3. Введи: `WSL: Connect to WSL`.
        4. VS Code подключится к Debian, после чего через File -> Open Folder можно открывать проекты (путь будет вида `\\wsl$\Debian\home\имя_пользователя\...`).


7️⃣ Основы Bash-скриптинга
     7.1  Ввод и вывод (read / echo)
        read <имя_переменной>      — считывает ввод пользователя.
        -p "Текст: "               — добавить подсказку перед вводом.
        -s                         — скрыть ввод (полезно для паролей).
        echo                       — вывод текста или переменных.
        echo $x                    — вывод значения переменной.
        echo $((x + y))            — выполнение арифметической операции и её вывод.
        nano                       — запускает консольный текстовый редактор в операционных системах UNIX и Linux.

     7.2 Перебор (Цикл For)
        read -p "Enter the upper limit: " n
        for (( i=1; i<=n; i++ ))
        do
            echo "$i"
        done

     7.3 Условия и сравнение (If / Else)
        - Для работы с числами можно использовать двойные круглые скобки `(( ))`, а для строк (и чисел) — квадратные `[ ]`.
        - Числа и строки (операторы `-eq`, `-ne`, `-gt`, `-lt`, `-ge`, `-le`):
                eq — равно (equal) ==
                ne — не равно (not equal) !=
                gt — больше  (greater than) >
                lt — меньше (less than) <
                ge — больше или равно (greater than or equal) >=
                le — меньше или равно (less than or equal) <=

        if [ "$x" -eq "$y" ]; then
        echo "Numbers are equal"
        elif [ "$x" -gt "$y" ]; then
        echo "First number is greater"
        else
        echo "Second number is greater"
        fi

       - Только для чисел (операторы `==`, `<`, `>`):*
        if (( x < y )); then
        echo "Second number is greater"
        elif (( x == y )); then
        echo "Numbers are equal"
        else
        echo "First number is greater"
        fi

        - Логические операторы:
           AND: `&&` (И)
           OR: `||` (ИЛИ)
                Использование внутри двойных скобок [[ ]]
                if [[ "$x" == "y" || "$x" == "Y" ]]; then ...

                Использование с одинарными скобками [ ]
                if [ "$x" = "y" ] && [ "$x" = "Y" ]; then ...
