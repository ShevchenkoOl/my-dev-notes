------------------------ SSH keys ------------------------------------
1️⃣ ls -al ~/.ssh  - проверяет или есть созданный ключи, выводит в детальном списке
ls ~/.ssh - тоже самое только результат будет в простом список

        Если мы работаем на Windows:
            dir $env:USERPROFILE\.ssh

Виды допуска:
1-я триада,Owner(Ты)                        rw-  Ты можешь читать (read) и записывать/менять (write). Но не можешь запускать как программу (x).
2-я триада,Group(Твоя команда/группа)       r--  Они могут только читать файл.
3-я триада,Others(Все остальные в системе)  r--  Любой другой пользователь может прочитать твой файл.
Расшифровка букв
            r (read) — чтение.
            w (write) — запись/изменение.
            x (execute) — выполнение (нужно для скриптов или программ).
            - — право отсутствует.
Примеры:
    -rwxrwxrwx (777) — «проходной двор». Все могут делать всё. Худший вариант для безопасности.
    -rwxr-xr-x (755) — стандарт для программ. Ты можешь всё, остальные только читать и запускать.
    -rw-rw-r-- (664) — часто встречается в общих проектах, где ты и твоя группа можете править файл.


2️⃣ ssh-keygen -t ed25519 -C "oleksii.shevchenko@gmail.com" - создание ключа, после ввода команды нужно три раза нажать Enter (в таком случаи мы все настройки оставим по умолчанию, но если нам нужно разделить или изменить название файлов, то можна указать: /home/oleksii/.ssh/id_ed25519_personal, Enter passphrase (empty for no passphrase): можно оставить пустым или придымать пароль, но в таком случаи пароль придётся вводить пароль при каждом push (или настраивать ssh-agent))

        ssh-keygen — сама программа (генерирует ключи).
        -t — флаг (сокращение от type), который говорит программе: «Я хочу выбрать тип алгоритма».
        ed25519 — конкретное имя алгоритма, который ты выбрал. Это самый современный, быстрый и безопасный стандарт шифрования для SSH-ключей.

3️⃣ в Debian\home\oleksii\.ssh - создастса 2 файла, один приватный, второй публичный с расширением .pub — это публичный ключ. Открываем блокнотом, копируем содержимое и вставляем в настройки GitHub. Открываем GitHub -> User Settings (иконка профиля справа вверху) -> SSH Keys -> New SSH key:
            -> title - название ПК который подкоючаеться к GitHub, например Laptop-Acer-Debian;
            -> Key type: Оставь Authentication Key (это стандарт);
            -> Key: Сюда нужно вставить текст из твоего файла id_ed25519.pub;
            -> Нажми Add SSH key.
        После добавления ключа рядом с ним может появиться кнопка "Configure SSO" (Single Sign-On). Если она появится, нажми её и выбери Authorize напротив названия "Economia"
📢 Если мы создаём SSH-ключи с использование ОС Windows, то кдючи будут храниться:  
    C:\Users\ТВОЙ_ПОЛЬЗОВАТЕЛЬ\.ssh

4️⃣ Проверка связи ssh -T git@github.com
Если он ответит: "Hi Oleksii! You've successfully authenticated", значит, мост между твоим компом и GitHub построен.

    Если мы работаем на Windows:
        ssh -V
    Ответ должен быть например: OpenSSH_for_Windows_9.5p2, LibreSSL 3.8.2

5️⃣ Для клонирование репозитория нужно использовать вкладку SSH, и дальше всё как обычно


⚠️ Когда сгенерировался ключ, открывает файл который без разширение:
    ls -l ~/.ssh/id_ed25519
        - и если ключ генерировался в Linux то должен увидеть -rw------- — всё супер, ничего делать не надо.
        Если вдруг видишь что-то другое (например, -rw-r--r--) — просто сам введи
        chmod 600 ~/.ssh/id_ed25519.
        - если ты генерировал в Windows, то там подробной информации о правах допуска не будет. Ничего страшного!


‼️‼️‼️Если мы используем 2 и больше фкаунтов на GitHub (робочий, личный и т.д.), тогда нам нужно к каждому аккаунту иметь свой SSH ключ и для того чтобы наш ПК не путался в ключах нужно:
- в папке .SSH создать файл config (можна без розширения);
- добавить следуюший текст:
    # Рабочий аккаунт
        Host github-work
        HostName github.com
        User git
        IdentityFile ~/.ssh/id_ed25519_work   - название файла где хрпниться робочий ключ

    # Личный аккаунт
        Host github-personal
        HostName github.com
        User git
        IdentityFile ~/.ssh/id_ed25519       - название файла где хрпниться личный ключ
- после того как мы сохранили файл нам нужно добавить его права, для ОС Linux - ОБЯЗАТЕЛЬНО, для Windows - нет:
    chmod 600 ~/.ssh/config
- Проверяем привязку озной из этих команд (в Windows для проверки используються теже команды):
    ssh -T git@github-work
    ssh -T git@github-personal
