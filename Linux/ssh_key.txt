------------------------ SSH keys ------------------------------------
1️⃣ ls -al ~/.ssh  - проверяет или есть созданный ключи, выводит в детальном списке
ls ~/.ssh - тоже самое только результат будет в простом список


Виды допуска:
1-я триада,Owner(Ты)                        rw-  Ты можешь читать (read) и записывать/менять (write). Но не можешь запускать как программу (x).
2-я триада,Group(Твоя команда/группа)       r--  Они могут только читать файл.
3-я триада,Others(Все остальные в системе)  r--  Любой другой пользователь может прочитать твой файл.
Расшифровка букв
            r (read) — чтение.
            w (write) — запись/изменение.
            x (execute) — выполнение (нужно для скриптов или программ).
            - — право отсутствует.
Примеры:
    -rwxrwxrwx (777) — «проходной двор». Все могут делать всё. Худший вариант для безопасности.
    -rwxr-xr-x (755) — стандарт для программ. Ты можешь всё, остальные только читать и запускать.
    -rw-rw-r-- (664) — часто встречается в общих проектах, где ты и твоя группа можете править файл.


2️⃣ ssh-keygen -t ed25519 -C "oleksii.shevchenko@gmail.com" - создание ключа, после ввода команды нудно три раза нажать Enter

        ssh-keygen — сама программа (генерирует ключи).
        -t — флаг (сокращение от type), который говорит программе: «Я хочу выбрать тип алгоритма».
        ed25519 — конкретное имя алгоритма, который ты выбрал. Это самый современный, быстрый и безопасный стандарт шифрования для SSH-ключей.

3️⃣ в Debian\home\oleksii\.ssh - создастса 2 файла, один приватный, второй публичный с расширением .pub — это публичный ключ. Открываем блокнотом, копируем содержимое и вставляем в настройки GitHub. Открываем GitHub -> User Settings (иконка профиля справа вверху) -> SSH Keys -> New SSH key:
            -> title - название ПК который подкоючаеться к GitHub, например Laptop-Acer-Debian;
            -> Key type: Оставь Authentication Key (это стандарт);
            -> Key: Сюда нужно вставить текст из твоего файла id_ed25519.pub;
            -> Нажми Add SSH key.
        После добавления ключа рядом с ним может появиться кнопка "Configure SSO" (Single Sign-On). Если она появится, нажми её и выбери Authorize напротив названия "Economia"

4️⃣ Проверка связи ssh -T git@github.com
Если он ответит: "Hi Oleksii! You've successfully authenticated", значит, мост между твоим компом и GitHub построен.

5️⃣ Для клонирование репозитория нужно использовать вкладку SSH, и дальше всё как обычно
