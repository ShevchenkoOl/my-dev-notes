Adminer — это легковесный инструмент с графическим интерфейсом (GUI) для управления базами данных прямо в браузере (аналог phpMyAdmin, только лучше и работает с PostgreSQL, MySQL и др.).
        Зачем нужен: Вместо того чтобы писать запросы в черной консоли (SELECT * FROM shop;), ты открываешь 127.0.0.2:8080, кликаешь на нужную таблицу мышкой, видишь все данные в виде красивой таблицы Excel, и можешь редактировать их двойным кликом. Это визуальный пульт управления твоей базой.

CDN — Content Delivery Network- Je to globální síť serverů rozmístěných po celém světě. Ukládají kopie souborů (například CSS, JS nebo obrázků) a poskytují je uživateli z nejbližšího serveru, aby se stránka načítala rychleji.

CI/CD - CI/CD (Continuous Integration / Continuous Deployment) Это автоматизация процессов, чтобы твой код попадал на сервер быстро и без боли. Состоит из двух частей:
        - CI (Непрерывная интеграция): Когда ты делаешь git push и создаешь Pull Request, специальная система (например, GitHub Actions или GitLab CI) автоматически скачивает твой код, проверяет его линтерами (Prettier, ESLint), прогоняет тесты и собирает проект. Цель: убедиться, что твой новый код не сломает всё остальное.

        - CD (Непрерывная доставка/развертывание): Если CI прошел успешно (тесты зеленые), система берет этот готовый код и автоматически заливает его на нужный сервер (на dev для тестов или на prod для пользователей). В вашем проекте за инфраструктуру и деплой отвечает AWS CDK.

CLI — Command Line Interface, rozhraní příkazového řádku

CRM (Customer Relationship Management) — это система управления взаимоотношениями с клиентами. В реалиях компании CRM — это сердце бизнеса. Именно там хранится информация о клиентах:
        Кто зарегистрировался на сайте.
        У кого куплена платная подписка (paywall) и когда она заканчивается.
        Какие email-рассылки человек читает.
        История оплат и активности пользователей.

DNS (Domain Name System) — это система, которая переводит понятные человеку буквенные адреса сайтов (например, yoursite.com) в цифровые IP-адреса, используемые компьютерами (например, 192.168.1.1). При настройке хостинга DNS связывает ваше доменное имя с сервером, где хранятся файлы вашего сайта.

Endpoint (Точка входа) — это конкретный адрес в интернете (URL), по которому программа ждет твоих команд. Например:
        http://localhost:3000/api/getArticles - это полный адрес апи запроса, а
        /api/getArticles — это и есть эндпоинт. Это "дверь", в которую ты стучишься, чтобы получить статьи.

EDP — это внутренняя аббревиатура Enterprise Data Platform (или Economia Data Platform), это огромное центральное "озеро данных" всей компании. Туда стекаются статьи, статистика просмотров, клики пользователей и реклама. Твой микросервис — это один из ручейков, который поставляет чистую воду (обработанные новости) в это огромное озеро EDP.

Fail-fast (буквально — «быстрый отказ» или «быстрая ошибка») означает принцип, при котором система или процесс немедленно прекращает работу при возникновении ошибки, вместо того чтобы пытаться продолжить выполнение. Его суть проста: если что-то идет не так, программа должна сломаться и выдать ошибку немедленно, прямо в тот самый момент, когда возникла проблема.

Feed Guid (Globally Unique Identifier) — это старый формат идентификатора. Обычно это длинная уникальная строка (текст), например 123e4567-e89b-12d3 или просто текстовый slug вроде realityMix.

IDE - Integrated Development Environment, это не просто текстовый редактор в виде блокнота, а целый набор инструментов в одной программе которые необзодимы для полноценного програмирования (Nejedná se jen o textový editor podobný poznámkovému bloku, ale o celou sadu nástrojů v jednom programu, které jsou nezbytné pro plnohodnotné programování):
                - Текстовый редактор (где пишем код) - Textový editor (kde píšeme kód).
                - Файловый менеджер (дерево папок проекта) - Správce souborů (strom složek projektu).
                - Терминал (командная строка для запуска тестов или сборки проекта) - Terminál (příkazový řádek pro spouštění testů nebo sestavování projektu).
                - Отладчик (Debugger) (инструмент для поиска ошибок) - Debugger (nástroj pro detekci chyb).
                - Система контроля версий (Git) (чтобы сохранять изменения в коде) - Systém pro správu verzí (Git) (pro ukládání změn kódu).

MBS (Merchant Business Services nebo Merchant and Banking Systems) 

Memcached (Мемкэшд) - Это сверхбыстрая краткосрочная память (кэш) для сервера. Когда бэкенд общается с внешними сервисами (например, с платежным шлюзом PayU), он получает временные ключи доступа (токены). Чтобы не запрашивать этот токен при каждой покупке заново (что долго), сервер сохраняет его в Memcached. Это как стикер на мониторе: данные лежат в оперативной памяти и доступны мгновенно.

MVP (Minimálně životaschopný produkt) je základní verze produktu, která obsahuje pouze nejdůležitější funkce potřebné k vyřešení klíčového problému uživatele a ověření obchodní hypotézy. Cílem MVP je co nejrychleji a s minimálními náklady otestovat nápad na skutečných uživatelích, získat zpětnou vazbu a zjistit, zda má smysl produkt dále rozvíjet.
Hlavní vlastnosti MVP:
            - Minimum funkcí, ale dostatečné pro použití
            - Rychlý vývoj a spuštění
            - Možnost sběru zpětné vazby
            - Základ pro další vylepšení

npm — Node Package Manager - Je to nástroj, který umožňuje instalovat, aktualizovat a spravovat knihovny a závislosti pro projekty v jazyce JavaScript.

npx — Node Package eXecute - Umožňuje spouštět balíčky přímo, aniž by bylo nutné je předem instalovat.

Payload (Пейлоад) - Полезная нагрузка. Это сами данные (тело запроса/ответа), которые передаются по API. Тот самый JSON в Postman с "id": 33, который ты нашел — это и есть пейлоад ответа сервера.

Pipeline (Пайплайн) - это пошаговый автоматический процесс, по которому твой код проходит путь от момента написания до попадания на рабочий сервер к пользователям. Это и есть та самая автоматическая "труба" шагов в CI/CD. Например: шаг 1 (собрать код) -> шаг 2 (проверить) -> шаг 3 (залить на сервер). Часто говорят: "Пайплайн упал" (значит, тесты не прошли или сборка сломалась).

RSS (Really Simple Syndication) — это технология, позволяющая легко получать новости с веб-сайтов, блогов или новостных серверов, которая автоматически доставляет обновления пользователю без необходимости вручную просматривать страницы.
Пример файла RSS в формате XML:
                        <rss>
                          <channel>
                                <title>Новости Economia</title>
                                  <item>
                                        <title>В Праге построят новый мост</title>
                                        <link>https://aktualne.cz/novy-most</link>
                                        <description>Краткий текст новости...</description>
                                  </item>
                                  <item>
                                        <title>Цены на недвижимость падают</title>
                                        <link>https://hn.cz/ceny</link>
                                        <description>Краткий текст новости...</description>
                                   </item>
                          </channel>
                        </rss>

SDK (Software Development Kit) — это набор инструментов от разработчика (в данном случае от Amazon/AWS), который позволяет твоему коду «общаться» с их сервисами.

SASS (Syntactically Awesome Stylesheets) - Je to CSS preprocesor, který přidává užitečné funkce, které v běžném CSS chybí — například proměnné, zanořování a mixiny.

SSR — Server-Side Rendering. Běžný React se vykresluje na straně klienta (v prohlížeči). Next.js ale dokáže vykreslit stránky na serveru – ještě předtím, než se pošlou do prohlížeče.
SSR znamená, že stránka se připraví už na serveru, takže se načte rychleji a je lepší pro SEO

SEO — Search Engine Optimization, optimalizace pro vyhledávače - Jedná se o soubor metod, které pomáhají webovým stránkám dosahovat vyšších pozici ve výsledcích vyhledávání.

SQS (Simple Queue Service) — это полностью управляемая служба очередей сообщений от Amazon Web Services (AWS), которая позволяет разделять и масштабировать микросервисы, распределенные системы и бессерверные приложения.

SQL (Structured Query Language) - язык структурированных запросов, стандарт для общения с реляционными базами.

Symfony (Симфони) - Это мощный каркас (фреймворк) для разработки на языке PHP. Представь, что ты строишь дом. Ты можешь делать кирпичи и пилить доски сам (чистый PHP), а можешь взять готовый каркас дома, где уже проведены трубы, есть фундамент и стены (Symfony).

Trunk-Based Development (TBD) — это стратегия управления версиями, при которой разработчики часто сливают небольшие изменения в одну общую ветку (называемую trunk, main или master) вместо создания множества долговременных параллельных веток. Это ускоряет интеграцию кода и предотвращает конфликты слияния (merge conflicts).

Workflow (Воркфлоу) - Рабочий процесс или конкретный скрипт автоматизации. Например, файл с инструкциями для GitHub Actions называется workflow. Также это может означать ваш процесс работы по Agile/Scrum.

UI (User Interface) — design uživatelského rozhraní Jde o to, jak produkt vypadá. UI designér se věnuje:
        1. Tlačítkům, menu, ikonám
        2. Barvám, písmům, mezerám
        3. Vizuálnímu stylu
        4. Animacím a přechodům
UI tedy dělá rozhraní krásným a vizuálně příjemným.

UX (User Experience) — design uživatelské zkušenosti Jde o to, jak produkt funguje a jaké pocity má člověk při jeho používání. UX designér se zaměřuje na:
        1. Logiku a strukturu aplikace nebo webu
        2. Pohodlnou navigaci
        3. Řešení problémů uživatele
        4. To, jak snadné a příjemné je dosáhnout cíle
Například UX designér promýšlí, kolik kroků je potřeba k dokončení objednávky v e-shopu, aby to bylo jednoduché a srozumitelné.

version: "0.0.0" в package.json - Semantic Versioning (SemVer) — стандарт MAJOR.MINOR.PATCH (например, 1.2.3):
                - PATCH (0.0.X): Исправление багов, опечатки, стили. Пример: 0.0.1 -> 0.0.2.
                - MINOR (0.X.0): Новая функциональность, которая не ломает старую. Пример: Добавил личный кабинет — 0.1.0.
                - MAJOR (X.0.0): Глобальные изменения, которые ломают обратную совместимость (например, полный переход на другую базу данных или фреймворк). Пример: Переезд на Next.js — 1.0.0.
