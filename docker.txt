1. Работа с образами (Images)
Образ — это "чертеж" или шаблон твоего приложения.
        docker pull <image_name> — скачать образ из Docker Hub.
    Пример: docker pull nginx (скачать официальный образ веб-сервера Nginx).

    docker images (или docker image ls) — посмотреть список всех скачанных образов на твоем ПК.
    docker build -t <name> . — собрать свой собственный образ из Dockerfile.
Пример: docker build -t my-app . (собрать образ с тегом "my-app" из текущей папки).

    docker rmi <image_id> — удалить образ.

2. Управление контейнерами (Containers)
Контейнер — это запущенный экземпляр образа (то, что реально работает).
        docker run <flags> <image> — создать и запустить контейнер.
    Пример: docker run -d -p 8080:80 --name web nginx

-d — запустить в фоновом режиме (detached).
-p 8080:80 — пробросить порт 8080 с твоего ПК на порт 80 внутри контейнера.
--name web — дать контейнеру понятное имя.
docker ps — список всех запущенных контейнеров.
docker ps -a — список вообще всех контейнеров (включая остановленные).
docker stop <name_or_id> — остановить контейнер.
docker start <name_or_id> — запустить ранее остановленный контейнер.
docker rm <name_or_id> — удалить контейнер.
docker exec -it <name> bash — "войти" внутрь работающего контейнера в терминал.

3. Очистка системы (Pruning)
Команды для удаления мусора, которые мы обсуждали в тесте.
docker system prune — удалить все остановленные контейнеры, неиспользуемые сети и "висячие" (dangling) образы.
docker container prune — удалить только остановленные контейнеры.
docker image prune — удалить только неиспользуемые образы.

4. Примеры использования в реальных задачах
Пример №1: Запуск базы данных PostgreSQL
Если тебе нужна база данных для разработки, не нужно её устанавливать в систему:

Bash
docker run --name my-db -e POSTGRES_PASSWORD=mysecret -d -p 5432:5432 postgres
Здесь -e задает переменную окружения (пароль).

Пример №2: Проверка ресурсов (Shared Memory)
Помнишь вопрос про 64 МБ? Вот как проверить это на практике:

Bash
docker run --rm alpine df -h /dev/shm
Эта команда запустит временный контейнер (--rm удалит его сразу после выполнения), покажет размер памяти и закроется.

Пример №3: Увеличение Shared Memory при запуске
Если твоему приложению (например, Chrome в Docker) мало 64 МБ:

Bash
docker run -d --shm-size=256m my-browser-app
Полезный "чит-код"
Если контейнеров и образов стало слишком много и они занимают всё место на диске, самая мощная команда для очистки:

Bash
docker system prune -a --volumes
Внимание: это удалит ВООБЩЕ ВСЁ, кроме запущенных контейнеров (включая все скачанные образы и данные в томах).
